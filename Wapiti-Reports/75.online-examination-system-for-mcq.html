Open Redirect

Description
Unvalidated redirects and forwards are possible when a web application accepts untrusted input that could cause the web application to redirect the request to a URL contained within untrusted input. By modifying untrusted URL input to a malicious site, an attacker may successfully launch a phishing scam and steal user credentials.

Vulnerability found in /75.online-examination-system-for-mcq/login.php

Description HTTP Request cURL command line 
Open Redirect via injection in the parameter q
GET /75.online-examination-system-for-mcq/login.php?q=https%3A%2F%2Fopenbugbounty.org%2F HTTP/1.1
Host: 192.168.10.7
curl "http://192.168.10.7/75.online-examination-system-for-mcq/login.php?q=https%3A%2F%2Fopenbugbounty.org%2F"
Solutions
Force all redirects to first go through a page notifying users that they are going off of your site, and have them click a link to confirm.

Cross Site Scripting

Description
Cross-site scripting (XSS) is a type of computer security vulnerability typically found in web applications which allow code injection by malicious web users into the web pages viewed by other users. Examples of such code include HTML code and client-side scripts.

Vulnerability found in /75.online-examination-system-for-mcq/index.php

Description HTTP Request cURL command line 
XSS vulnerability found via injection in the parameter w
GET /75.online-examination-system-for-mcq/index.php?w=%3C%2Fscript%3E%3CScRiPt%3Ealert%28%27w0gf5kz3an%27%29%3C%2FsCrIpT%3E HTTP/1.1
Host: 192.168.10.7
curl "http://192.168.10.7/75.online-examination-system-for-mcq/index.php?w=%3C%2Fscript%3E%3CScRiPt%3Ealert%28%27w0gf5kz3an%27%29%3C%2FsCrIpT%3E"
Solutions
The best way to protect a web application from XSS attacks is ensure that the application performs validation of all headers, cookies, query strings, form fields, and hidden fields. Encoding user supplied output in the server side can also defeat XSS vulnerabilities by preventing inserted scripts from being transmitted to users in an executable form. Applications can gain significant protection from javascript based attacks by converting the following characters in all generated output to the appropriate HTML entity encoding:<, >, &, ', (, ), #, %, ; , +, -
